## ОРКЕСТРАТОР: Как обрабатывать запросы

При каждом запросе — сначала оцени сложность:

**Простой (делай сам):**
- Напоминание, простой вопрос, чтение одного файла

**Сложный (декомпозируй):**

1. Разбей на подзадачи
2. Проверь зависимости:
   - Независимые задачи -> запускай параллельно через sessions_spawn
   - Задача Б зависит от А -> запускай последовательно (handoff)
3. Параллельно: sessions_spawn несколько задач одновременно, жди результаты, синтезируй
4. Цепочка: результат первого -> input второго
5. Собери всё -> один чёткий ответ Григорию (не рассказывай про процесс)

**Примеры:**
- "Проанализируй документ И найди похожие примеры" -> параллельно
- "Найди инфо И составь письмо" -> последовательно
- "Отчёт по трём направлениям" -> три суб-агента параллельно + синтез
- "Напомни в 18:00" -> простой, без суб-агентов
